# Security Policy

本项目是医疗助手工程项目，涉及用户身份、医院凭证、医疗咨询和预约挂号等敏感场景。安全策略以“最小权限、显式确认、可审计、默认保守”为原则。

## 敏感信息

不要提交以下内容：

- `.env`、`project/.env`
- LLM API key、JWT secret、数据库密码
- 医院 MCP access token、refresh token、cookie、个人医院账号
- 真实患者姓名、手机号、身份证号、就诊卡号、病历号
- 本地运行数据、日志、上传缓存和数据库 dump

仓库已通过 `.gitignore` 排除常见运行产物和本地配置。如果新增敏感路径，请同步更新 `.gitignore`。

## 运行态安全边界

- 所有 `/api/*` 接口默认需要 Bearer Token。
- 管理类接口需要 admin role。
- 用户只能访问自己的会话、医院绑定和凭证。
- 同一会话通过 session lock 避免并发写乱状态。
- Redis 可用时用于跨 worker 的运行态保护。

## MCP 与挂号安全

预约、取消、改约属于高风险动作，必须满足：

- 已明确医院，或用户只绑定一家可用医院。
- 多医院未明确时必须澄清，不允许默认选择第一家。
- 执行前必须生成预约预览。
- 用户明确确认后才调用医院 MCP 工具。
- 确认阶段必须沿用 pending payload 中的 `hospital_code`。
- MCP 工具映射只在指定医院 namespace 下查找工具，不允许 fallback 到其他医院。

## 医疗回答安全

本项目不是医疗器械，不提供诊断结论，不替代医生。医疗回答应：

- 尽量基于知识库证据。
- 对证据不足的问题给出保守提示。
- 对急危重症状提醒及时就医。
- 避免给出个体化用药剂量、处方替代和诊断断言。

## 漏洞处理

如果发现安全问题，请优先记录：

- 影响范围
- 复现步骤
- 涉及接口或模块
- 是否包含敏感信息泄露
- 建议修复方式

在真实部署前，应补充生产级审计日志、密钥轮换、HTTPS、数据库备份、最小权限数据库账号和访问告警。
